## [Talk] IHM critiques : quand les tâches utilisateur testent le code

Celia Martinie

Utiliser des "task models" pour décrire les parcours utilisateurs de façon formelle. Et les lier au code applicatif pour soit piloter le code appli par les tâches afin de vérifier que tout fonctionne comme attendu, ou à l'inverse utiliser l'application et obtenir le parcours des tâches.
A voir comment ça s'intègre dans des projets classiques.
